How does Fast R-CNN improve over R-CNN?

1. Shared Feature Map

In R-CNN, each crop is fed through the CNN backbone individually (e.g. 100 crops = 100 forwards). Fast R-CNN passes the **entire image** through the backbone once, generating a shared feature map. This removes the main computational bottleneck.

2. RoI Pooling Layer

The RoI Pooling layer maps arbitrary-sized region proposal coordinates onto the shared feature map, extracting a fixed $7 \times 7$ feature vector for each proposal. This allows fully connected classification and regression heads to run instantly in parallel.

3. Bounding Box Regression

Along with a class probability, the model predicts bounding box coordinates offsets. Instead of relying on raw proposal coordinates, Fast R-CNN **regresses** (scales and shifts) the initial proposals to tightly fit the borders of the digits.
